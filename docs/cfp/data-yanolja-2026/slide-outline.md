# Slide Outline

## 발표 길이

- 총 30분
- 목표 난이도: Lv2
- 발표 톤: 실무 문제 중심, 수식은 숨기지 않되 쉽게 읽히게 설명

## 발표의 중심 약속

베이지안 최적화는 하이퍼파라미터 튜닝 도구를 넘어, 상황에 따라 달라지는 비싼 의사결정 실험을 적은 시행착오로 최적화하는 방법이라는 점을 전달한다.

## 발표 흐름 요약

1. 문제 제기: 왜 A/B 테스트만으로는 부족한가
2. 베이지안 최적화의 기본 구조
3. CRM 정책 최적화 예제
4. Constraint / 연령·성별별 정책 최적화 확장
5. Ax / BoTorch와 실무 적용 포인트

## 슬라이드 구성

### 1. 오프닝

- 목표 시간: 2분
- 핵심 메시지: 베이지안 최적화는 아직도 하이퍼파라미터 튜닝용으로만 소비되는 경우가 많다
- 보여줄 것:
  - 실무 의사결정 예시 카드 3개
  - 가격 정책, CRM 정책, 운영 파라미터 최적화 예시
- 청중에게 남길 질문:
  - "이건 모델 튜닝 문제일까, 의사결정 최적화 문제일까?"

### 2. 왜 A/B 테스트만으로는 부족한가

- 목표 시간: 3분
- 핵심 메시지: 단일 비교 실험은 정책 공간 탐색 문제를 직접 풀지 못한다
- 보여줄 것:
  - A/B 테스트의 강점
  - 정책 파라미터가 2개 이상일 때 조합 수가 급격히 늘어나는 모습
  - `할인율 × 발송 시점` 예시
- 시각화:
  - 단일 A/B 비교와 연속 정책 탐색의 차이

### 3. 우리가 실제로 푸는 문제

- 목표 시간: 2분
- 핵심 메시지: 우리는 결국 `x* = argmax_x f(x)`를 푼다
- 수식:
  - `x* = argmax_x f(x)`
- 설명 포인트:
  - `x`는 정책 파라미터 조합
  - `f(x)`는 정책을 실행했을 때 얻는 기대 성과
- 시각화:
  - 2D 정책 공간 위의 숨겨진 목적함수 surface

### 4. Gaussian Process를 함수의 분포로 이해하기

- 목표 시간: 4분
- 핵심 메시지: 정답 함수를 모르기 때문에 함수 자체에 대한 분포를 둔다
- 수식:
  - `f ~ GP(m, k)`
- 설명 포인트:
  - prior sample path
  - 관측이 들어오면 posterior가 어떻게 바뀌는지
  - 평균과 불확실성을 동시에 본다는 의미
- 시각화:
  - prior 함수 샘플
  - posterior mean + uncertainty band

### 5. noisy observation

- 목표 시간: 2분
- 핵심 메시지: 실험 결과는 진짜 성과와 노이즈가 섞인 관측이다
- 수식:
  - `y = f(x) + ε`
- 설명 포인트:
  - 마케팅 실험, 운영 실험, 정책 실험은 본질적으로 noisy하다
  - 그래서 적은 실험으로도 uncertainty를 반영하는 접근이 중요하다
- 시각화:
  - 같은 점에서도 결과가 흔들리는 산점도

### 6. 다음 실험점은 어떻게 고르나

- 목표 시간: 3분
- 핵심 메시지: 잘 될 것 같은 곳과 아직 불확실한 곳 사이에서 균형을 잡는다
- 수식:
  - `x_next = argmax_x [μ(x) + βσ(x)]`
- 설명 포인트:
  - exploitation
  - exploration
  - 베이지안 최적화가 "감으로 찍는 탐색"이 아닌 이유
- 시각화:
  - mean, uncertainty, acquisition 직관 그림

### 7. CRM 합성 시뮬레이터 소개

- 목표 시간: 3분
- 핵심 메시지: 공개 가능한 합성 시뮬레이터만으로도 정책 최적화 구조를 충분히 보여줄 수 있다
- 설정:
  - 정책 파라미터: 할인율, 발송 시점
  - 목적함수: `incremental profit - coupon cost - send cost`
  - 결과 변수: 반응률, 구매 전환, 비용, 7일 무접속 이탈률
- 시각화:
  - 입력 -> 고객 반응 -> 비용/성과 -> objective 계산 흐름도

### 8. 결과 1: 제약 없는 최적화

- 목표 시간: 2분
- 핵심 메시지: 적은 실험으로 좋은 정책 구간을 좁혀간다
- 보여줄 것:
  - objective heatmap
  - iteration trajectory
  - random/grid와 비교한 탐색 효율
- 시각화:
  - 2D heatmap 위 실험점 이동 경로

### 9. 결과 2: 제약 있는 최적화

- 목표 시간: 2분
- 핵심 메시지: 수익만 보는 최적점과 실제로 운영 가능한 최적점은 다를 수 있다
- 수식:
  - `max_x f(x)  subject to  g(x) <= 0`
- 설명 포인트:
  - `g(x)`: 메시지 수신 후 7일 무접속 이탈률
  - feasible / infeasible 구분
- 시각화:
  - feasible 영역이 겹쳐진 heatmap

### 10. 연령·성별별 정책 최적화

- 목표 시간: 3분
- 핵심 메시지: 연령과 성별이 달라지면 최적 정책도 달라질 수 있다
- 수식:
  - `x*(c) = argmax_x f(x, c)`
- 설명 포인트:
  - `20대 여성`, `20대 남성`, `30대 여성`, `30대 남성`처럼 그룹별 최적점이 달라질 수 있다
  - "전역 최적점"보다 "그룹별 최적점"이 실무적으로 더 중요할 수 있다
- 시각화:
  - 연령/성별 그룹별 다른 objective surface 또는 다른 optima

### 11. 그룹을 완전히 따로 볼 필요는 없다

- 목표 시간: 2분
- 핵심 메시지: 닮은 연령/성별 그룹은 정보를 공유하며 더 적은 실험으로도 안정적인 탐색이 가능할 수 있다
- 수식:
  - `x*(c) = argmax_x f(x, c)`
- 설명 포인트:
  - 연령/성별 그룹을 완전히 따로 최적화하지 않아도 된다
  - 닮은 그룹은 유사한 구조를 공유할 수 있다
  - 그룹별 데이터가 적을수록 이런 관점이 더 중요해진다
- 시각화:
  - 연령/성별 그룹이 연결된 구조도

### 12. 이런 문제에도 적용할 수 있다

- 목표 시간: 1분
- 핵심 메시지: 중요한 것은 라이브러리가 아니라 문제를 어떻게 파라미터화하느냐이다
- 짧게 언급할 파라미터 조합:
  - 할인율 × 쿠폰 유효기간
  - 할인율 × 발송 비율
  - 할인율 × uplift threshold
- 추가 실무 예시:
  - 가격 정책
  - 예산 배분
  - 운영 파라미터 조정

### 13. Ax / BoTorch로 시작하기

- 목표 시간: 1분
- 핵심 메시지: 구현 장벽은 생각보다 높지 않다
- 설명 포인트:
  - `Ax`: 빠르게 시작하기 좋은 실무형 래퍼
  - `BoTorch`: 더 유연한 커스텀 최적화 도구
  - 라이브러리보다 중요한 것은 objective, constraint, group context 정의
- 시각화:
  - Ax와 BoTorch의 역할 구분 다이어그램

### 14. 마무리

- 목표 시간: 1분
- 핵심 메시지:
  - 베이지안 최적화는 하이퍼파라미터 튜닝만의 도구가 아니다
  - A/B 테스트 이후의 정책 탐색 문제를 다루는 도구가 될 수 있다
  - 실무에서 중요한 것은 objective, constraint, group context를 구조화하는 능력이다
- 시각화:
  - objective / constraint / context 3개 키워드 정리

## 발표 중 강조할 문장

- "예측만 잘한다고 의사결정이 자동으로 좋아지지는 않습니다."
- "A/B 테스트는 매우 강력하지만, 정책 공간을 직접 최적화해주지는 않습니다."
- "베이지안 최적화의 핵심은 다음 점을 잘 찍는 것이고, 그 출발점은 문제 구조화입니다."

## 부록 또는 QnA에서 다룰 수 있는 고급 포인트

- discrete context를 함께 다루는 구조와 ICM 계열 아이디어
- context embedding을 활용하는 LCEMGP 계열 아이디어
- discrete + continuous 입력을 함께 다루는 mixed joint GP 관점
- 실제 noisy 환경에서 qNEI, batch BO, constraint modeling으로 확장하는 흐름
