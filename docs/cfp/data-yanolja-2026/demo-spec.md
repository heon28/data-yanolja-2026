# Demo Spec

## 목적

외부 공개 가능한 완전 합성 CRM 시뮬레이터를 통해 베이지안 최적화의 기본 구조와 확장을 시연합니다. 데모의 목표는 "복잡한 현실을 그대로 재현하는 것"이 아니라, 정책 최적화 문제를 직관적으로 설명할 수 있는 최소한의 구조를 만드는 것입니다.

## 발표에서 데모가 맡는 역할

- `할인율 × 발송 시점`이라는 2차원 정책 공간을 통해 베이지안 최적화의 직관을 보여준다
- objective, constraint, group context가 모두 들어가는 문제 구조를 시각적으로 설명한다
- random/grid 대비 베이지안 최적화가 적은 실험으로 유망한 영역을 좁혀가는 과정을 보여준다
- 연령/성별별 최적화로 어떻게 자연스럽게 확장되는지 연결한다

## 기본 설정

### 정책 파라미터

- `discount_rate`
  - 예: 0% ~ 30%
- `send_timing`
  - 예: 기준 이벤트 이후 0일 ~ 7일

### 데모용 권장 범위

- `discount_rate`: `0.00` ~ `0.30`
- `send_timing`: `0` ~ `7`
- 초기 관측점: 6~8개
- 총 BO evaluation: 20~25회
- 비교 baseline:
  - random search 20~25회
  - grid search 7 x 8 또는 유사 밀도

### 맥락 변수 후보

- 연령대
- 성별
- 요일
- 채널

## 목적함수

발표에서 사용할 핵심 목적함수는 아래와 같습니다.

```text
Objective = incremental profit - coupon cost - send cost
```

보다 구체적으로는 다음과 같이 해석합니다.

```text
f(x, c) = uplift_revenue(x, c) - coupon_cost(x) - send_cost(x)
```

- `x`: 정책 파라미터 조합
- `c`: 연령대, 성별, 채널 같은 group context
- `uplift_revenue`: 메시지로 인해 추가로 발생한 순증분 매출
- `coupon_cost`: 할인으로 인해 지불한 비용
- `send_cost`: 발송 자체에 드는 비용

발표에서는 이 식을 복잡하게 전개하지 않고, "정책이 성과와 비용을 함께 바꾼다"는 수준으로 설명합니다.

## 제약 조건

수익만 높은 정책이 항상 좋은 정책은 아니므로, 아래 제약 조건을 함께 둡니다.

```text
g(x, c) = 7일 무접속 이탈률 - 허용 임계값
```

그리고 최적화 문제는 다음처럼 설명합니다.

```text
max_x f(x, c)  subject to  g(x, c) <= 0
```

발표용 해석:

- 할인율을 과하게 높이거나 발송 시점을 공격적으로 잡으면 단기 반응은 좋아질 수 있다
- 하지만 지나친 메시지는 장기 무접속 이탈률을 높일 수 있다
- 따라서 실제 운영에서는 objective와 constraint를 함께 봐야 한다

## 시뮬레이터 가정

데모는 설명이 잘 되는 형태를 우선합니다. 아래와 같은 가정을 둡니다.

- 너무 낮은 할인율은 반응을 거의 만들지 못한다
- 너무 높은 할인율은 반응은 증가시키지만 비용이 커져 이익이 줄어든다
- 너무 이른 발송 또는 너무 늦은 발송은 효과가 떨어진다
- 연령/성별 그룹에 따라 할인율 민감도와 발송 시점 민감도가 다르다
- 과도한 정책은 단기 성과를 높일 수 있지만 장기 이탈률을 증가시킨다
- 따라서 objective surface는 매끄럽지만 단조롭지 않고, constraint surface는 다른 모양을 가진다

## 권장 합성 시나리오

발표용으로는 연령/성별 그룹을 4개만 두는 것이 가장 직관적입니다.

### 20대 여성

- 할인율 변화에 반응이 크다
- 발송 시점은 중간 정도로 영향이 있다

### 20대 남성

- 할인율보다 발송 시점에 조금 더 민감하다
- 너무 이른 발송에서 이탈률 페널티가 빨리 커진다

### 30대 여성

- 중간 할인율에서 안정적인 반응을 만든다
- 너무 늦은 발송에서 효과가 빠르게 줄어든다

### 30대 남성

- 큰 할인 없이도 특정 타이밍에서 반응이 올라간다
- 공격적인 정책에는 상대적으로 보수적인 반응을 보인다

이 구조를 쓰면 연령/성별별로 최적점이 달라진다는 점과, 닮은 그룹 간 정보를 공유하는 직관을 함께 설명하기 쉽습니다.

## 권장 함수 형태

구현은 자유롭게 하되, 발표용으로는 아래 형태를 추천합니다.

### 반응률

- 할인율에 대해 완만한 증가 후 포화
- 발송 시점에 대해 중간 지점에서 최대가 되는 종 모양
- 연령/성별 그룹별로 peak 위치 또는 민감도를 다르게 설정

### 비용

- 쿠폰 비용은 할인율에 비례하게 증가
- 발송 비용은 상수 또는 아주 약한 선형 항으로 둔다

### 이탈률 constraint

- 너무 이르거나 늦은 발송 시점에서 증가
- 지나치게 공격적인 할인 정책에서 추가 증가
- 일부 연령/성별 그룹이 timing penalty에 더 민감하게 반응하도록 설정

핵심은 수학적 정교함보다, `unconstrained optimum`과 `constrained optimum`이 분명히 다르게 보이도록 surface를 설계하는 것입니다.

## 기대하는 시각적 결과

### 결과 1: 제약 없는 objective surface

- `할인율 × 발송 시점` 평면에서 heatmap 생성
- optimum 주변이 보이도록 매끄러운 2D surface 설계
- BO iteration이 유망 영역으로 모여드는 경로 시각화

### 결과 2: constraint surface

- 같은 2D 공간에서 이탈률 constraint를 별도 heatmap 또는 contour로 표현
- feasible / infeasible 영역을 명확히 구분
- unconstrained optimum과 constrained optimum의 차이를 눈으로 확인 가능하게 구성

### 결과 3: group context comparison

- `20대 여성`, `20대 남성`, `30대 여성`, `30대 남성`이 서로 다른 최적점을 가지도록 설계
- "모든 고객에게 같은 정책이 최적이 아님"을 직관적으로 보여줌

### 결과 4: group sharing intuition

- 연령/성별 그룹을 완전히 독립적으로 두지 않고 일부 구조를 공유하는 그림을 가정
- 완전히 따로 푸는 것보다 정보를 공유할 때 탐색이 더 안정적일 수 있음을 설명

## 발표용 추천 비교 화면

- random search vs Bayesian Optimization
- unconstrained vs constrained optimum
- 연령/성별별 optimum 비교
- 그룹을 따로 볼 때 vs 정보를 공유할 때의 직관 비교

## 발표 중 짧게 언급할 추가 파라미터 조합

이 항목들은 데모의 중심은 아니지만, 같은 프레임으로 확장 가능한 예시로 짧게 언급합니다.

- 할인율 × 쿠폰 유효기간
- 할인율 × 발송 비율
- 할인율 × uplift threshold (advanced)

## 실무 문제로 확장할 때의 예시

CRM 외에도 아래와 같은 문제를 같은 관점으로 재해석할 수 있음을 발표에서 짧게 언급합니다.

- 가격 정책 최적화
- 메시징 빈도와 시점 최적화
- 채널별 예산 배분
- 운영 파라미터 조정
- 실험 자원 배분 문제

## 구현 메모

- 발표용 결과는 직관이 우선이므로 지나치게 복잡한 행동 모형은 피합니다
- 2차원 공간에서 optimum이 분명히 보이도록 함수 구조를 설계합니다
- constraint가 실제로 해를 바꾸는 예시가 나오도록 파라미터를 맞춥니다
- 연령/성별 group context 차이가 선명하게 보이도록 설계합니다
- Ax / BoTorch 언급은 후반부에서 짧게 다루고, 데모 설명의 중심은 문제 구조화에 둡니다
